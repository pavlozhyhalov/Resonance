# Resonance

Персональний застосунок: музика по частотах (реальні аудіофайли + YouTube), дихання за методом Віма Хофа, практика холоду, календар активності з аналізом, завдання й винагороди. Дані синхронізуються через **окремий** проєкт Supabase (не повʼязаний із сайтом клану).

---

## Оновлення застосунку (коли я присилаю нову версію)

1. Розпакуй новий `resonance-app.zip`.
2. На github.com у репозиторії `Resonance` → **Add file → Upload files**.
3. Перетягни **весь вміст** розпакованої папки у вікно (з підпапками `css`, `js`, `icons`, `audio`).
4. Унизу — **Commit changes**. GitHub оновить змінені файли й додасть нові.
5. Через 1–2 хвилини онови сайт у браузері (на iPhone — закрий і відкрий ярлик).

> Перший раз тобі також треба завантажити нову папку **`audio/`** (звуки) та нові файли `js/icons.js`, `js/pages/analytics.js` — вони підтягнуться автоматично при завантаженні всієї папки.

---

## Перший вхід

Тепер це **окремий** проєкт Supabase, тож реєстрація вже не конфліктує з кланом.

1. Відкрий сайт → **«Немає акаунту? Зареєструватися»**.
2. Введи пошту й пароль (від 6 символів) → **Створити акаунт**.
3. Підтверди акаунт за листом на пошті → повернись і **увійди**.
4. Той самий акаунт на телефоні й компʼютері = спільні дані.

## Звук

- Усі звуки — це **реальні аудіофайли** в папці `audio/`, які грають всередині застосунку. На iPhone вони звучать навіть якщо ввімкнено «беззвучно», бо йдуть через медіаканал.
- **Перший дотик до екрана вмикає звук** (вимога Safari). Тапни будь-де — і далі все звучить.
- Для **бінаурал-ритмів** (Delta/Theta/Alpha/Beta/Gamma/Шуман) потрібні навушники.

### Додати свою музику
1. Поклади свій файл, напр. `my-track.mp3`, у папку `audio/`.
2. У файлі `js/audio.js` знайди список `SOUNDSCAPES` і додай рядок:
   `{ id:"mytrack", label:"Моя музика", file:"my-track.mp3" },`
3. Залий зміни (Commit). Трек зʼявиться у виборі музики для дихання й холоду.

(Скажи мені — і я зроблю окрему вкладку «Моя музика» з кількома треками.)

## Розділи

- **Частоти** — тапни частоту → програвач. Вкладка *Аудіо* грає файл; *YouTube* — ембієнт усередині сторінки. Гучність — повзунком. Сесія від 20 с іде в календар.
- **Дихання (Вім Хоф)** — раунди, кількість дихань (повзунок), темп, відновлення, музика, гучність (повзунок), підказки. Звукові сигнали на вдиху/видиху та гонг на затримці. Затримку завершуєш **подвійним тапом по екрану або кнопкою**. Якщо зупинити достроково — усе одно запишеться в календар.
- **Холод** — душ або ванна, тривалість, музика, гучність. Круговий таймер. Запис у календар.
- **Календар** — кольорові мітки активності; кнопка **«Аналіз»** веде до діаграм (метрика, період, тип практики — налаштовуються).
- **Цілі** — завдання дають бали; **1 хвилина практики = 1 бал**; бали обмінюються на винагороди.

## Технічно
- Бекенд: окремий проєкт Supabase `Resonance` (eu-west-1). Таблиці `sessions`, `tasks`, `task_completions`, `rewards`, `settings` із RLS — дані бачиш лише ти.
- Ключ у `js/config.js` — публічний (publishable), безпечний у відкритому репозиторії.
- Чесно про частоти: наукова база є лише в бінаурал-ритмів (попередня). Solfeggio й 432 Гц — традиція без доказів (позначено «Без доказів»).
