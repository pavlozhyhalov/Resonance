# Resonance

Персональний застосунок: музика по частотах (синтез + YouTube), дихання за методом Віма Хофа, практика холоду, календар активності, завдання й винагороди. Дані синхронізуються через Supabase після входу.

---

## 1. Що всередині

```
resonance-app/
├─ index.html              ← головний файл
├─ manifest.webmanifest    ← налаштування PWA (іконка, назва)
├─ css/styles.css          ← усі стилі й теми розділів
├─ icons/                  ← іконки застосунку
└─ js/
   ├─ config.js            ← адреса й ключ Supabase (уже вписані)
   ├─ store.js             ← база даних + вхід
   ├─ audio.js             ← синтез тонів, бінаурал, звуки, YouTube
   ├─ data.js              ← список частот
   ├─ router.js, ui.js, app.js
   └─ pages/               ← головна, частоти, дихання, холод, календар, цілі
```

Бекенд Supabase уже налаштовано: створено таблиці `sessions`, `tasks`, `task_completions`, `rewards`, `settings` із захистом по користувачу (RLS). Нічого додатково в Supabase робити не треба.

## 2. Як викласти на GitHub Pages (через GitHub Desktop)

1. **Створи репозиторій.** На github.com → `New repository` → назва, напр. `resonance`. Зроби його **Public** (для безкоштовного Pages). Натисни *Create*.
2. **Клонуй у GitHub Desktop.** `File → Clone repository` → обери `resonance` → клонуй у будь-яку папку.
3. **Скопіюй файли.** Поклади **вміст** папки `resonance-app/` (тобто `index.html`, `css/`, `js/`, `icons/`, `manifest.webmanifest`) у корінь клонованої папки. Важливо: `index.html` має лежати в корені репозиторію, не у вкладеній папці.
4. **Закоміть і запуш.** У GitHub Desktop унизу зліва напиши коротко (напр. `init`), натисни *Commit to main*, потім *Push origin*.
5. **Увімкни Pages.** На github.com у репозиторії → `Settings → Pages` → *Source* обери `Deploy from a branch` → гілка `main`, папка `/ (root)` → *Save*.
6. За 1–2 хвилини зʼявиться адреса виду `https://ТВІЙ_НІК.github.io/resonance/`. Відкрий її.

Будь-які майбутні зміни: відредагував файл → у GitHub Desktop *Commit* → *Push*. Сайт оновиться сам.

## 3. Встановити на iPhone (як застосунок)

1. Відкрий адресу сайту в **Safari** (саме Safari, не Chrome).
2. Натисни кнопку **Поділитися** (квадрат зі стрілкою вгору).
3. Обери **«На екран Початок»** (Add to Home Screen) → *Додати*.
4. На робочому столі зʼявиться іконка Resonance — відкривається на весь екран, без адресного рядка.

На Android: меню браузера (Chrome) → *Встановити застосунок* / *Add to Home screen*.

## 4. Перший вхід

- При першому відкритті — екран входу. Натисни *«Немає акаунту? Зареєструватися»*, введи пошту й пароль (від 6 символів).
- Supabase за замовчуванням надсилає лист підтвердження. Відкрий лист, підтверди, потім увійди тією ж поштою. Далі вхід зберігається — щоразу вводити не треба.
- Той самий акаунт на телефоні й компʼютері = спільні дані (сесії, календар, бали).

## 5. Як користуватися

- **Частоти.** Тапни на частоту → плеєр. *Синтез* грає точний тон офлайн; для бінаурал-ритмів (Delta, Theta, Alpha, Beta, Gamma, Шуман) **потрібні навушники**. *YouTube* — ембієнт-музика всередині сторінки. Сесія від 20 с автоматично потрапляє в календар.
- **Дихання (Вім Хоф).** Задай раунди, кількість дихань, темп, відновлення, музику й підказки → *Почати*. Під час затримки **подвійний тап** по екрану завершує її. У кінці — час кожного раунду, усе зберігається.
- **Холод.** Обери душ або ванну, тривалість → *Зануритись*. Круговий таймер із паузою. Запис у календар.
- **Календар.** Кольорові крапки = типи активності за день. Тап по дню — деталі, можна видаляти записи.
- **Цілі.** Створюй щоденні завдання (дають бали) і винагороди (купуються за бали).

## Важливі примітки

- **Чесно про частоти.** Наукова база є лише в бінаурал-ритмів (і то попередня). Solfeggio-тони (174–963 Гц) і 432 Гц — традиція/маркетинг без доказів; у застосунку це позначено бейджем «Без доказів».
- **YouTube-вкладка** показує пошуковий результат у вбудованому плеєрі; інколи Google вимагає тап для старту або показує рекламу. Синтез — основний надійний режим.
- **Безпека даних.** Ключ Supabase у `config.js` — публічний (publishable), його не страшно тримати у відкритому репозиторії: доступ до даних обмежено правилами RLS на рівні бази.
- **Звук** запускається після першого дотику до екрана (вимога браузерів) — це нормально.
